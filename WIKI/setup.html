🚀 Projekt Opsætning

Følg disse trin for at få Flutter Calories appen kørende på din lokale maskine.

📋 Forudsætninger

Før du begynder, sørg for at du har følgende installeret:

  • Flutter SDK: Version 3.22 eller nyere. Projektet bruger Dart SDK 3.8.0+. Installer Flutter
  • Git: For at klone repository'et. Installer Git
  • IDE (Anbefalet):
    • Visual Studio Code med Flutter extension.
    • Android Studio med Flutter og Dart plugins.

📂 Klon Repository

Klon projektet fra GitHub (eller din versionskontrol platform):

git clone https://github.com/your-repo/calories-app.git
cd calories-app
Bemærk: Erstat https://github.com/your-repo/calories-app.git med den faktiske URL til repository'et.

📦 Installer Dependencies

Når du har klonet repository'et, naviger til projektmappen og kør følgende kommando for at hente alle nødvendige pakker:

flutter pub get

Dette vil downloade alle dependencies specificeret i pubspec.yaml filen.

🛠️ Kør Build Runner (Hvis Nødvendigt)

Projektet bruger kodegenerering (f.eks. for Freezed). Hvis du støder på fejl relateret til genererede filer, eller efter at have trukket ændringer der involverer modeller, skal du køre build_runner:

flutter pub run build_runner build --delete-conflicting-outputs

Dette kan tage et par minutter første gang, eller hvis der er mange filer at generere.

▶️ Kør Appen

Sørg for at du har en emulator kørende, eller en fysisk enhed tilsluttet.

  1. Vælg enhed: Brug enhedslisten i din IDE (VS Code eller Android Studio) for at vælge din mål-enhed/emulator.
  2. Kør appen: 
    flutter run

Appen skulle nu bygge og starte på din valgte enhed/emulator.

Tip: Hvis du har flere enheder tilsluttet, kan du specificere en enhed med flutter run -d <device_id>. Kør flutter devices for at se en liste over tilgængelige enheder.

💡 IDE Specifik Opsætning

Visual Studio Code

  • Installer Flutter og Dart extensions.
  • Åbn projektmappen i VS Code.
  • Brug F5 tasten eller "Run > Start Debugging" for at starte appen.
  • VS Code vil automatisk bruge launch.json hvis den findes, for specifikke startkonfigurationer.

Android Studio

  • Installer Flutter og Dart plugins (File > Settings > Plugins).
  • Åbn projektet ved at vælge "Open an Existing Project" og naviger til projektmappen.
  • Vælg en enhed fra enheds-dropdown menuen.
  • Klik på "Run" knappen (den grønne trekant).

🔧 Almindelige Opsætningsproblemer

  • Flutter version ikke opfyldt:

    Hvis du ser en fejl om at din Flutter version er for gammel, kør:

    flutter upgrade
  • Manglende Android SDK/Command-line tools:

    Kør flutter doctor for at diagnosticere problemer med din Flutter installation, Android toolchain, Xcode, etc. Følg anbefalingerne fra flutter doctor.

  • Problemer med genererede filer (Freezed, etc.):

    Sørg for at køre flutter pub run build_runner build --delete-conflicting-outputs efter at have opdateret pakker eller trukket ændringer der påvirker modeller.

  • Platform-specifikke fejl (iOS/Android):
    • For iOS: Naviger til ios/ mappen og kør pod install eller pod update.
    • For Android: Tjek at du har den korrekte version af JDK, og at dine Android SDK build tools er opdaterede. Prøv at køre "Invalidate Caches / Restart" i Android Studio.
Tip: Se Troubleshooting siden for flere specifikke løsninger på problemer, der er opstået under udviklingen.
