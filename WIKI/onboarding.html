Step-by-Step Flow

1. 🏠 Welcome Step

OnboardingStep.welcome
├── App branding & logo
├── Feature highlights
├── "Kom i gang" button
└── No validation required

2. 👤 Personal Info Step

OnboardingStep.personalInfo
├── Name input (required)
├── Date of birth (DD/MM/YYYY format)
│   ├── Three separate input fields
│   ├── Real-time age calculation
│   └── Date validation
├── Gender selection (male/female)
└── Validation: name.isNotEmpty && dateOfBirth != null && gender != null

3. 📏 Physical Info Step

OnboardingStep.physicalInfo
├── Height slider (120-220 cm)
├── Current weight slider (30-200 kg)
├── Target weight slider (30-200 kg)
├── Real-time BMI calculation
└── Validation: heightCm > 0 && currentWeightKg > 0

4. 🎯 Goals Step

OnboardingStep.goals
├── Goal type selection
│   ├── Weight loss
│   ├── Weight maintenance  
│   ├── Weight gain
│   └── Muscle gain
├── Activity level selection
│   ├── Sedentary
│   ├── Lightly active
│   ├── Moderately active
│   ├── Very active
│   └── Extra active
├── Weekly goal adjustment
└── Validation: goalType != null && activityLevel != null

5. 📋 Summary Step

OnboardingStep.summary
├── Complete profile overview
├── Calculated BMR/TDEE values
├── Target calories display
├── Macronutrient breakdown
├── Edit buttons for each section
└── "Fuldfør" button

6. ✅ Completion Step

OnboardingStep.completed
├── Success animation
├── Welcome message
├── Profile saved confirmation
└── Navigation to main app

Auto-Save Funktionalitet

Onboarding implementerer intelligent auto-save for at forhindre data tab:

// Auto-save triggered på alle input changes
void updateName(String name) {
  state = state.copyWith(
    userProfile: state.userProfile.copyWith(name: name),
  );
  _autoSaveProgress(); // ✅ Auto-save
}

void updateDateOfBirth(DateTime dateOfBirth) {
  state = state.copyWith(
    userProfile: state.userProfile.copyWith(dateOfBirth: dateOfBirth),
  );
  _calculateTargets(); // Recalculate BMR/TDEE
  _autoSaveProgress(); // ✅ Auto-save
}

Data Calculations

🧮 BMR (Basal Metabolic Rate)

Bruger Mifflin-St Jeor equation for præcise beregninger:

// For mænd
BMR = (10 × weight_kg) + (6.25 × height_cm) - (5 × age) + 5

// For kvinder  
BMR = (10 × weight_kg) + (6.25 × height_cm) - (5 × age) - 161

🔥 TDEE (Total Daily Energy Expenditure)

TDEE = BMR × Activity Multiplier

Activity Levels:
├── Sedentary: BMR × 1.2
├── Lightly Active: BMR × 1.375
├── Moderately Active: BMR × 1.55
├── Very Active: BMR × 1.725
└── Extra Active: BMR × 1.9

🍎 Target Calories

// Goal adjustments (1 kg = 7700 kcal)
Weight Loss: TDEE - (weeklyGoalKg × 7700 ÷ 7)
Weight Gain: TDEE + (weeklyGoalKg × 7700 ÷ 7)
Maintenance: TDEE (no adjustment)

// Safety constraints: 1200-4000 kcal

🥗 Macronutrient Targets

Macronutrient Distribution:
├── Protein: 25% of calories (÷ 4 kcal/g)
├── Fat: 30% of calories (÷ 9 kcal/g)
└── Carbs: 45% of calories (÷ 4 kcal/g)

State Management

📊 OnboardingState Structure

@freezed
class OnboardingState with _$OnboardingState {
  const factory OnboardingState({
    @Default(OnboardingStep.welcome) OnboardingStep currentStep,
    @Default(UserProfileModel()) UserProfileModel userProfile,
    @Default(false) bool isLoading,
    @Default(false) bool hasError,
    String? errorMessage,
    @Default(false) bool isEditingFromSummary,
  }) = _OnboardingState;

  // Computed properties
  bool get canProceedToNext => /* validation logic */;
  double get progress => currentStep.index / (OnboardingStep.values.length - 2);
}

🔄 Step Navigation

// Forward navigation with validation
void nextStep() {
  if (!state.canProceedToNext) return;

  final nextStep = switch (state.currentStep) {
    OnboardingStep.welcome => OnboardingStep.personalInfo,
    OnboardingStep.personalInfo => OnboardingStep.physicalInfo,
    OnboardingStep.physicalInfo => OnboardingStep.goals,
    OnboardingStep.goals => OnboardingStep.summary,
    OnboardingStep.summary => OnboardingStep.completed,
    OnboardingStep.completed => OnboardingStep.completed,
  };

  state = state.copyWith(currentStep: nextStep);
}

Data Persistence

💾 Storage Strategy

Storage Layers:
├── Partial Progress: SharedPreferences
│   ├── Key: 'user_profile_partial'
│   ├── Auto-save on every change
│   └── Loaded on app restart
│
└── Final Profile: SharedPreferences  
    ├── Key: 'user_profile'
    ├── Saved on completion
    └── Cleared partial progress

Future _autoSaveProgress() async {

Performance Optimizations

// Debounced auto-save to prevent excessive storage calls
Timer? _saveTimer;

Future _autoSaveProgress() async {
  _saveTimer?.cancel();
  _saveTimer = Timer(const Duration(milliseconds: 500), () async {
    await OnboardingStorageService.savePartialProgress(state.userProfile);
  });
}
