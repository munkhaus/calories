🔌 API Reference & Eksterne Services

Oversigt over eksterne API'er og services brugt i Flutter Calories appen.

🤖 Google Gemini AI

Status: Aktuelt brugt, men planlagt til udfasning.
Se LLM Elimination Plan for detaljer.

Google Gemini AI bruges til avancerede funktioner som fødevaregenkendelse via billeder og intelligent søgning i fødevaredatabasen.

  • Officiel Dokumentation: Google AI for Developers
  • SDK brugt: google_generative_ai Dart pakke.

1. LLMFoodService (Fødevaresøgning)

Formål: Leverer søgefunktionalitet i appens fødevaredatabase ved hjælp af naturlig sprogbehandling.

Implementering: lib/features/food_database/infrastructure/llm_food_service.dart

Status: Aktiv, men skal erstattes af Open Food Facts API.

API Endpoints (Konceptuelt via SDK): Bruger Gemini 1.5 Flash model til tekst-til-tekst generering baseret på søgeforespørgsler.

Autentificering: Kræver API nøgle konfigureret i projektet (håndteres typisk via environment variabler eller en sikker konfigurationsmekanisme - ikke specificeret yderligere her).

Data Modeller: Input er en tekststreng (søgeforespørgsel). Output er en liste af fødevareobjekter (f.eks. OnlineFoodResult DTO) parset fra Gemini's svar.

2. GeminiService (Billedanalyse)

Formål: Analyserer billeder af mad for at identificere fødevarer og estimere næringsindhold.

Implementering: lib/features/food_logging/infrastructure/gemini_service.dart

Status: Aktiv, men skal erstattes af forbedret manuel indtastning og potentielt andre non-LLM billedanalyseværktøjer i fremtiden.

API Endpoints (Konceptuelt via SDK): Bruger Gemini modeller (f.eks. Gemini 1.5 Flash eller Pro Vision) til multimodal analyse (tekst og billede input).

Autentificering: Kræver API nøgle, ligesom LLMFoodService.

Data Modeller: Input er et eller flere billeder (og evt. tekst prompts). Output er struktureret data om de genkendte fødevarer (f.eks. FoodAnalysisResult DTO).

🥦 Open Food Facts API

Status: Foreslået erstatning for LLMFoodService.
Se Detaljer i LLM Elimination Plan.

Open Food Facts er en åben og crowdsourced database med information om fødevarer fra hele verden. Det er et gratis alternativ til LLM-baseret fødevaresøgning.

  • Officiel Dokumentation: Open Food Facts API Docs
  • SDK/Pakke: Kan bruges direkte via HTTP requests (f.eks. med http pakken) eller via en community-vedligeholdt Dart pakke, hvis en passende findes. llm-elimination.html foreslår direkte HTTP.

OpenFoodFactsService (Fødevaresøgning)

Formål: Søge efter fødevarer i Open Food Facts databasen.

Implementering (Foreslået): lib/features/food_database/infrastructure/open_food_facts_service.dart (skal oprettes).

API Endpoints:

  • Søgning: https://world.openfoodfacts.org/api/v2/search?search_terms=QUERY&page_size=20&json=true

Autentificering: Ingen påkrævet for de fleste læse-operationer, men en User-Agent header anbefales.

Data Modeller: Input er en søgeforespørgsel. Output er et JSON objekt indeholdende en liste af produkter. Se Open Food Facts API dokumentationen for den præcise struktur. DTOs (Data Transfer Objects) bør oprettes i appen for at mappe disse svar.

Eksempel på respons (forkortet):

{
  "count": 12345,
  "page": 1,
  "products": [
    {
      "product_name": "Eksempel Produkt",
      "nutriments": {
        "energy-kcal_100g": 250,
        "proteins_100g": 10,
        "fat_100g": 5,
        "carbohydrates_100g": 40
      },
      // ... mange flere felter
    }
  ]
}
